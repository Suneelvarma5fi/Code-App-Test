# Dataverse Form — Power Apps Code App

A minimal single-page application (SPA) that inserts data into a **Dataverse `contact` table** directly from a browser form. Built as a **Power Apps Code App** using React + TypeScript + Vite and the `@microsoft/power-apps` SDK.

The goal is **not** the UI — it is **establishing the data bridge** between a code-first SPA and Dataverse, and performing data operations (Create, Read, Delete) via PAC CLI generated services.

---

## How the Data Bridge Works

```
User fills form
  → React Component (DataEntryForm)
    → Custom Hook (useDataverseForm)
      → Generated Service (ContactsService.create)
        → @microsoft/power-apps SDK (getClient)
          → Power Platform Connector Layer
            → Dataverse table (contact)
```

- **Generated services** are created by `pac code add-data-source`. You never write raw API calls.
- At runtime inside Power Apps, the SDK handles **authentication** (Entra ID), **DLP policies**, and **connector routing** automatically.
- Locally, you test via the **Local Play URL** that the Vite plugin prints to the terminal.

## Quick Start

### 1. Clone and install

```bash
git clone https://github.com/Suneelvarma5fi/Code-App-Test.git
cd Code-App-Test
npm install
```

### 2. Authenticate to Power Platform

```bash
pac auth create --environment 0481c8d6-5203-4308-a186-824cdf4f517e
```

Follow the browser prompt to sign in with your Entra ID. Replace the environment ID above with your own if using a different environment.

### 3. Re-generate the data source (⚠️ CRITICAL STEP)

The repo contains manually-written generated files (created because the original developer was behind a corporate proxy). **You must delete them and let the CLI regenerate properly** — this registers the data source with the Power Apps runtime:

```bash
# Windows (CMD)
rmdir /s /q src\generated
rmdir /s /q .power

# macOS / Linux
rm -rf src/generated .power
```

Then run:

```bash
pac code init --displayname "Dataverse Form App"
pac code add-data-source -a dataverse -t contact
```

This generates:
- `.power/schemas/appschemas/dataSourcesInfo.ts` — runtime data source registration
- `src/generated/models/ContactsModel.ts` — TypeScript entity types
- `src/generated/services/ContactsService.ts` — CRUD methods (`create`, `getAll`, `update`, `delete`)

### 4. Verify import paths

After generation, confirm that `src/hooks/useDataverseForm.ts` imports still match the generated file paths:

```typescript
import { ContactsService } from "../generated/services/ContactsService";
import type { Contacts } from "../generated/models/ContactsModel";
```

If the CLI generated them at different paths, update the imports accordingly.

### 5. Build and deploy to Power Apps

```bash
npm run build
pac code push
```

This publishes the app to your environment. The CLI prints a URL like:

```
App has successfully been pushed!
You can test the app at: https://apps.powerapps.com/play/e/<env-id>/app/<app-id>
```

Open that URL in your browser, or find the app in [make.powerapps.com](https://make.powerapps.com).

### 6. Test the app

- Fill in First Name and Last Name (required), plus optional Email, Phone, Job Title
- Click **Submit** — the record should appear in the list below
- Click **Delete** on any record to remove it
- Click **Refresh** to reload from Dataverse

> **⚠️ VPN / Corporate Proxy Note**
>
> If `pac code add-data-source` fails with DNS errors like `ENOTFOUND *.environment.api.powerplatform.com`, your machine can't reach Power Platform APIs directly. Run steps 2–5 from a machine **without** a restrictive proxy, or configure `HTTP_PROXY` / `HTTPS_PROXY` environment variables.

## Dataverse Table Mapping

Default target: **`contact`** (exists in every Dataverse environment).

| Form Field | Dataverse Column | Type | Required |
|---|---|---|---|
| First Name | `firstname` | string | ✅ |
| Last Name | `lastname` | string | ✅ |
| Email | `emailaddress1` | string | |
| Phone | `telephone1` | string | |
| Job Title | `jobtitle` | string | |

### Use a different table

To target a custom or different Dataverse table:

```bash
pac code add-data-source -a dataverse -t <table-logical-name>
```

Then update the form fields in `DataEntryForm.tsx` and the service calls in `useDataverseForm.ts` to match the new table's columns.

## Troubleshooting

| Issue | Solution |
|---|---|
| `pac` not found | Install [Power Platform CLI](https://learn.microsoft.com/en-us/power-platform/developer/cli/introduction) |
| Auth failed | Run `pac auth create` and sign in |
| CORS errors locally | Use the **Local Play URL** from terminal output, not `localhost` directly |
| `src/generated/` missing | Run `pac code add-data-source -a dataverse -t contact` |
| Build errors after generate | Delete `node_modules`, run `npm install` again |
| App stuck on "fetching" | Check `power.config.json` exists and has a valid `environmentId` |
